    介绍¶

    本文介绍用 Mathematica 实现非交换代数的计算，例如Lie algebra, quantum group 及其表示。

    市面上能找到的Lie algebra package 多数是针对semisimple Lie algebra 特化的， SuperLie package 看起来可以实现我们需要的功能，有兴趣的小伙伴可以尝试一下。在 Headrick的Virasoro algebra notebook 基础上，我们可以总结出两种方法加载代数关系，其一是他采用的延迟赋值SetDelayed，其二是ReplaceRepeated，根据MMA自带的文档，前者在计算中实际上就是采用了后者。我个人的经验是后者计算更加可控。 代码放在GitHub - Yuriever/noncommutative-algebra 。 MMA提供了非交换乘法函数 NonCommutativeMultiply，它的中缀符号是**，内置的属性(attributes)为， In[]:= NonCommutativeMultiply // Attributes Out[]= {Flat, OneIdentity, Protected} 其中flat 即结合律，它会自动摊平所有的括号。注意我们对于Lie algebra \mathfrak{g} 和其weight module的计算是在universal enveloping algebra U(\mathfrak{g}) 中进行的，quantum group 可以粗糙的理解为U(\mathfrak{g}) 的形变，这些例子遵循结合律。修改属性的函数是SetAttributes和ClearAttributes。方便起见，我们添加Listable属性， Unprotect[NonCommutativeMultiply]; SetAttributes[NonCommutativeMultiply,Listable]; 接下来自然的问题是，什么表达式是非交换的？我们用一个列表algebraOperator来存储非交换的算子，然后用FreeQ来判定一个表达式是交换的标量(C-number)还是算子(Q-number)， (声明代数) algebraOperator={}; (判断是否为通常的C-number) cnumberQ[expr_]:=And@@(FreeQ[expr,#]&/@algebraOperator); 这里#和& 是MMA的纯函数写法，/@是Map，不熟悉的读者可以直接跳过，接受cnumberQ[expr]当expr是C-number时返回True即可。

    我们可以指定一系列规则来加载非交换乘法的关系，然后用函数ReplaceRepeated进行化简。在此之前，非交换乘法需要满足标量乘法和分配律，在一些应用中需要定义算子的共轭或者类似的anti-involution，这些基础规则不依赖具体的代数。用列表algebraRelationInternal来存储以上基本规则，把用户可操作的代数规则存储到列表algebraRelation中，把所有的规则存储在列表algebraRelationAll中， algebraRelation={}; (用algebraRelationInternal来隐藏基本规则) algebraRelationInternal=basicAlgebraRelationJoinbasicConjugationRelation; Protect[algebraRelationInternal]; (初始化algebraRelationAll) algebraRelationAll=algebraRelationInternal;

    basicAlgebraRelation={ x_y_:>x y/;cnumberQ[x]||cnumberQ[y], (x_?cnumberQ y_)**z_:>x y**z, x_(y_?cnumberQ z_):>y x**z, (x_+y_)z_:>x**z+y**z, x_(y_+z_):>x**y+x**z }; basicConjugationRelation={ conjugate[x_**y_]:>conjugate[y]**conjugate[x], conjugate[x_+y_]:>conjugate[x]+conjugate[y], conjugate[x_ y_]:>conjugate[x]conjugate[y], conjugate[x_?cnumberQ]:>Conjugate[x] }; 然后定义一个函数封装ReplaceRepeated（//.是其后缀写法）， algebraSimplify[x_]:=x//.algebraRelationAll//Simplify; SetAttributes[algebraSimplify,Listable]; 为了使计算结果可读，我们可以定义额外的打印规则， algebraPrintRule={};

    algebraPrint[x_]:=x//.algebraPrintRule; SetAttributes[algebraPrint,Listable]; 最后是列表操作函数，用来添加和清除规则， addOperator; addRelation; addPrintRule;

    clearOperator; clearRelation; clearPrintRule; clearAlgebra; 以及一些其他常用函数， commutator; jacobiIdentity; adjoint; Adjoint; innerProduct; 为了保证ReplaceRepeated的结果收敛，我们需要确定最后表达式中算子的优先级，定义一个Lie algebra上的序关系，即根据Poincaré–Birkhoff–Witt定理，挑出一组universal enveloping algebra中的基。 例子1：以Virasoro algebra为例，我们指定算子 L_n ，以及一些态， {L, vac, primary, descendant} // addOperator 然后把下标大的交换到右边， {L[n_] ** L[m_] :> L[m] ** L[n] + (n - m) L[n + m] + If[n + m == 0, c1 (n^3 - n)/12 , 0] /; n > m, } // addRelation
