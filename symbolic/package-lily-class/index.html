Package lily`class`

The packages lily`class` and lily`list` are designed to manage a specific form of class composition illustrated in the following flow chart, implemented by Association and DownValues respectively.

In this page we focus on the former since there are several advantages over the latter: Association is immutable and faster to access, consumes less memory, and requires no symbol management.

Instance initializing and updating

Motivation and designing ideas

A finitely presented algebra (or monoid, module, ...) consists of lists of generators operator and relations relation. Besides, a list of rules printing is also required when formatting the output. From above we can abstract the following object

  • class: algebra

  • instance attribute: name, operator, relation, printing

  • method: define, unset, reset, add_elements, delete_elements

Taking only operator as an example, it can be implemented as

The class algebra in Python and Mathematica 
class Algebra():

    # define
    def __init__(self, name):
        self.name = name
        self.operator = set()

    # unset
    def __del__(self):
        print(f'The algebra {self.name} has been unset.')

    def reset(self):
        print(f'The algebra {self.name} has been reset.')
        self.operator = set()

    def add_elements(self, elements):
        self.operator = self.operator.union(elements)
        return self.operator

    def delete_elements(self, elements):
        self.operator = self.operator.difference(elements)
        return self.operator

algCheck["ifNotDefined"][name_] :=
    If[ KeyExistsQ[alg,name]==False,
        Print@StringTemplate["The algebra `` has not been defined."][name];
        Abort[]
    ];
algCheck["ifHasDefined"][name_] :=
    If[ KeyExistsQ[alg,name]==True,
        Print@StringTemplate["The algebra `` has been defined."][name];
        Abort[]
    ];

algDefine[name_] :=
    Module[ {},
        algCheck["ifHasDefined"][name];
        AssociateTo[alg,name-><|"operator"->{}|>];
    ];

algUnset[name_] :=
    Module[ {},
        algCheck["ifNotDefined"][name];
        Print@StringTemplate["The algebra `` has been unset."][name];
        KeyDropFrom[alg,name];
    ];

algReset[name_] :=
    Module[ {},
        algCheck["ifNotDefined"][name];
        Print@StringTemplate["The algebra `` has been reset."][name];
        AssociateTo[alg,name-><|"operator"->{}|>];
    ];

algAdd[name_][elements_] :=
    Module[ {list},
        algCheck["ifNotDefined"][name];
        list = Union[alg[name,"operator"],elements];
        AssociateTo[alg[name],"operator"->list];
        alg[name]
    ];

algDelete[name_][elements_] :=
    Module[ {list},
        algCheck["ifNotDefined"][name];
        list = Complement[alg[name,"operator"],elements];
        AssociateTo[alg[name],"operator"->list];
        alg[name]
    ];

Several different algebras correspond to different list triples and can be compounded into one algebra

  • class: compounded algebra

  • instance attribute: algebra_list

  • method: define, unset, protect

This leads to an abstraction of the three-layered structure: compounded algebra->algebra->{operator,relation,printing}, hereinafter referred as class->instance->member respectively.

There are two designing ideas as follows:

  • DownValues - member is the core, class is the UpValues of the member, and instance is the DownValues of the member. Following this idea we get the package lily`list`:

    classOf[member1] ^= class;

member1[instance1] = {};
member1[instance2] = {};
...

  • Association - the data is stored into nested associations. Following this idea we get the package lily`class`:

    class = <|
    instance1-><|member1->{},member2->{},...|>,
    instance2-><|member1->{},member2->{},...|>,
    ...
|>

Attributes

Public attributes

  • classData[class_] - store the data of class_.

    • "instanceCommonData" - store the common data of instances, determining the behaviors of instanceDefine and instanceReset.

    • "instanceExtraData" - store the extra data of instances, contributing to instanceDefaultData.

    • "instanceProperty" - store the properties of instances, only affected by instanceDefine and instanceUnset.

    • "instanceDefaultList" - store the list of instances as default, changed by instanceDefault and instanceUnset, determining instanceDefaultData.

    • "instanceAdd" - cores of method instanceAdd according to member structures.

    • "instanceDelete" - cores of method instanceDelete according to member structures.

    • "memberStructureIdentity" - identities of member structures under instanceAdd and instanceDelete.

    • "memberStructureUsage" - descriptions of member structures.

    • "memberStructureData" - list of member structures.

    • "memberList" - list of members.

    • "isClassProtected" - state of class protection.

  • instanceData[class_] - store the data of instances with members.

  • instanceDefaultData[class_] - store the data of default instance of class. After instance operations the default values will be updated.

Private attributes

  • memberStructureInternal - pre-defined data structures of members, including:

    • "boolean" - Boolean value: add is Or and delete is And.

    • "string" - string: add is StringJoin and delete is StringDelete.

    • "symbol" - symbol: add is replacing and delete is replacing with Null.

    • "assocUnsorted" - unsorted association without duplicates.

    • "listUnsorted" - unsorted list allowing duplicates.

    • "listSorted" - sorted list allowing duplicates.

    • "setUnsorted" - unsorted set without duplicates.

    • "setSorted" - sorted set without duplicates.

  • memberStructure - store the member structures.

  • memberStructureQ[structure_] - check whether the structure is defined.

  • memberStructureDefine[structure_,assoc_] - define a new data structure of member.

  • memberStructureUnset[structure_] - unset a non-internal member structure.

Methods

Class constructors

  • classDefine[class_,_,_,_] - define and initiate the class.

    • classDefine[class_,memberList_,structureList_,commonValueList_,extraValueList_] - the arguments are name of class, names of members, data structures of members, common values of members and extra values of members.

    • classDefine[class_,memberList_,structureList_] - a shortcut of arguments. The common and extra values will be supplemented according to the "memberStructureIdentity" of member structureList_.

  • classDefineQ[class_] - check whether the class is defined.

  • classProtect[class_,_?BooleanQ] - protect the defined class against classUnset. The protected class will not be unset when reloading the package.

  • classUnset[class_] - unset the class if defined and unprotected. When loading the package, unprotected class will be unset.

Public methods

  • instanceDefineQ[class_,instance_|instanceList_] - check whether the instance is defined, or split the list of instances into defined and undefined.

  • instanceDefine[class_,instanceList_,property_:Null] - define the instances.

  • instanceDefault[class_,instanceList_] - set the instances into default.

  • instanceReset[class_,instanceList_] - reset the instances.

  • instanceUnset[class_,instanceList_] - unset the instances, and update the default instance list.

  • instanceAdd[class_,instanceList_,memberAssoc_] - add elementlist to the instances. The memberAssoc_ will be association-ized, hence rule, list of rules, association of rules are accepted.

  • instanceDelete[class_,instanceList_,memberAssoc_] - delete elementlist from the instances. The memberAssoc_ will be association-ized, hence rule, list of rules, association of rules are accepted.

Private methods

  • instanceDefineCheck[_][__] - the inputs will be checked by this private method before calling public methods:

    • instanceDefineCheck["classAbortUndef"][class_]

    • instanceDefineCheck["instanceReportUndefAndReturnDef"][class_,instanceList_]

    • instanceDefineCheck["instanceReportDefAndReturnUndef"][class_,instanceList_]

    • instanceDefineCheck["memberAbortUndef"][class_,memberList_]

  • instanceDefaultUpdate[class_] - the default instance will be updated by this private method after calling public methods of default, reset, unset, add and delete.

Decorators

The instance methods can be intercepted by the decorators instancePreIntercept, instancePostIntercept, see the definition of instance*`kernel. The APIs are

instancePreIntercept["instanceDefaultUpdate"][class_,defaultInstance_];
instancePostIntercept["instanceDefaultUpdate"][class_,defaultInstance_];

instancePreIntercept["instanceDefine"][class_,instance_,property_];
instancePostIntercept["instanceDefine"][class_,instance_,property_];

instancePreIntercept["instanceDefine"][class_,instanceList_];
instancePostIntercept["instanceDefine"][class_,instanceList_];

instancePreIntercept["instanceReset"][class_,instance_];
instancePostIntercept["instanceReset"][class_,instance_];

instancePreIntercept["instanceUnset"][class_,instance_];
instancePostIntercept["instanceUnset"][class_,instance_];

instancePreIntercept["instanceAdd"][class_,instance_,member_,list_];
instancePostIntercept["instanceAdd"][class_,instance_,member_,list_];

instancePreIntercept["instanceDelete"][class_,instance_,member_,list_];
instancePostIntercept["instanceDelete"][class_,instance_,member_,list_];
