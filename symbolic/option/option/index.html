Optional arguments

本页面介绍 Mathematica 的选项功能。

基本用法

以下称 OptionsPattern[] 为输入选项，称 Options[] 为默认选项。例如：

Example

child//Options = {"a"->0};

child[OptionsPattern[]] :=
    OptionValue["a"];

child[{{"a"->1,"a"->2}}]

Out[] = 1

选项的继承

此种用法适用于将简单函数从下到上的组合为复杂函数。例如：

Example

parent//Options = {
    Splice@Options@child,
    "b"->x
};

parent[opts:OptionsPattern[]] :=
    child@FilterRules[{opts},Options[child]];

SetOptions[parent,"a"->1]

parent[]

Out[] = {a->1,b->x}

Out[] = 0

parent//Options = {
    Splice@Options@child,
    "b"->x
};

parent[opts:OptionsPattern[]] :=
    child@FilterRules[{opts,Options@parent},Options[child]];

SetOptions[parent,"a"->1]

parent[]

Out[] = {a->1,b->x}

Out[] = 1

此处 FilterRules 会过滤出 child 接受的选项。

在错误的例子中，若用 SetOptions 更新 parent 的默认选项，可以发现该更新未传递给 child。其原因在于键 "a" 未出现在 parent 的输入选项中，因此 child 调用了自身的默认选项 "a"->0。

在正确的例子中，我们用 {opts,Options@parent} 手动合并了 parent 的输入选项与默认选项，再将其传递给 child。

非局域选项的指定

上述继承的用法有时稍显笨重，特别是在顶层函数仅用来分离参数与选项，或检测参数类型，而非实现核心功能的时候。例如：

Example

foo//Options = {"a"->0};

foo[args___] :=
    With[ {sep = ArgumentsOptions[foo[args],{1,2},<|"Head"->HoldComplete,"OptionsMode"->"Shortest"|>]},
        ifoo[sep]/;!FailureQ[sep]
    ];

foo[x,"a"->1]

foo[x,"b"->y]

Out[] = ifoo[{HoldComplete[x],HoldComplete["a"->1]}]

Out[] = ifoo[{HoldComplete[x,"b"->y],HoldComplete[]}]

可见 "a"->1 被正确的识别为了选项，而未出现在 foo 的默认选项中的 "b"->y 被识别为了参数。

接下来部分实现 ifoo 的功能，

Code

ifoo[{HoldComplete[x_],HoldComplete[opts___]}] :=
    x+OptionValue[foo,{opts},"a"];

此处 OptionValue[foo,{opts},"a"] 自动接受了 foo 的输入选项与默认选项。

Code

SetOptions[foo,"a"->1];

foo[x]

foo[x,"a"->0]

Out[] = 1+x

Out[] = x

类似的，OptionsPattern[foo] 亦可以实现此种非局域的选项指定。

注意事项

  • OptionValue 优先搜索输入选项，其次搜索默认选项。

  • 输入选项的键 (key) 应首先出现在默认选项中。

  • 若选项的键出现重复，前面的优先级更高。这与关联 (association) 的行为相反，关联的重复键中后面的优先级更高。

  • 需注意，若函数接受的参数中含有列表，FilterRules 返回的列表可能会被匹配为列表而非输入选项。

  • 此外，输入选项接受序列、列表和嵌套列表，后者为文档未指明的行为。默认选项也接受嵌套列表，在特定场景会报错，不影响功能。FilterRules 的两个参数接受列表和嵌套列表，返回列表。
