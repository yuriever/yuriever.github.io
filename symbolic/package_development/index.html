Paclet and package development

扩展包结构可概括为：

p.wl
(*用Get强制重新读取扩展包p0.wl...*)
Get/@{"p0`",...}

(*设定$Context为p`，Needs扩展包p1.wl...*)
BeginPackage["p`",{"p1`",...}];

(*重载扩展包时清除上次的定义*)
dataDelete["..."];(*from lily`list`*)
Unprotect@@Names["p`*"];
ClearAll@@Names["p`*"];

(*声明公共函数*)
foo::usage = "";

(*设定$Context为p`Private`*)
Begin["`Private`"];

(*函数定义*)
foo = ...;

(*结束当前$Context，恢复为p`*)
End[];

(*保护所有未指名的符号*)
protectAllExcept[
    "foo1"
];

(*恢复$Context，并添加p`至$ContextPath*)
EndPackage[];

Some tips

Function type

  • 常数 (constant)，存储变量的列表以及列表操作函数；

  • 判别函数 (questing function)，用以界定变量的类型和范围，因此定义尽量完备；

  • 主体函数 (body function)，实现该扩展包的各种功能。为防止意外计算，需要定义再去定义，遵循自上到下的书写规则；

  • 打印函数 (format)，区分返回值和打印，分离函数之间传递的值和输出结果。

Naming

  • 同一类功能的函数以相同的前缀命名。

  • 判别函数的命名后缀为 Q。

  • 局部变量的名称为小写字母，全局变量 foo 的局部拷贝命名为 $$foo，重复模式的名称为单词复数 foos__，列表模式的名称为 foolist_，helper 函数命名为 foo`foo1。

Others

Decoupling

减少复杂的不同功能的函数、不同扩展包之间的关联。

Helper functions

复杂的主体函数应分层，最外层提供用户接口，若干内层的 helper 函数实现不同功能，尽可能使函数的依赖关系为 tree-like 的结构，为了避免重复判定条件，仅在最外层使用判别函数。

Functions with attributes HoldAll

如无必要，不使用 HoldAll 等属性，以防阻止期待的计算。

Sub-packages and auto-completion

子扩展包的自动补全问题：母扩展包 p` 下的子扩展包 p`p1` 中的公共函数不会被自动补全。可见讨论 Sub-packages and auto-completion。

解决方案：将母扩展包解耦并重命名为 p0`，在 p0` 中启用子扩展包 p`p1`。

OwnValues is bad

公共符号应避免使用 OwnValues: foo=... 而使用 foo[]=...。

Dummy rules

ReplaceAll 访问表达式的策略是自上到下的，匹配完的子表达式不再被匹配，因此：一方面，规则 {x_foo:>x,...} 可以防止 Head 为 foo 的子表达式被匹配，称为哑规则；另一方面，其余的规则应避免此类不改变表达式的行为，以防止子表达式被遮蔽。
