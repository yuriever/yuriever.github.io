    扩展包开发¶

    扩展包结构可概括为：

    BeginPackage["p`", {"p1`", ...}];

(*重载扩展包时清除上次的定义*)
Unprotect @@ Names["p`*"];
ClearAll @@ Names["p`*"];

(*声明公共函数*)
foo::usage = "";

Begin["`Private`"];

foo = ...;

End[];
EndPackage[];

    扩展包中的函数可以分为下列：

    • 常数，例如主体函数中的搜索次数等等；

    • 存储变量的列表以及列表操作函数；

    • 判别函数，用以界定变量的类型和范围，因此定义尽量完备；

    • 主体函数，实现该扩展包的各种功能。为防止意外计算，需要定义再去定义，遵循自上到下的书写规则；

    • 打印函数，区分返回值和打印，分离函数之间传递的值和输出结果。

    一些经验：

    • 脱耦合。减少不同功能的函数、不同扩展包之间的关联。
    • 减少HoldAll 等属性的使用，以防阻止期待的计算。特别的，判别函数不应带有此类属性。
    • 子扩展包的自动补全。母扩展包 p` 下的子扩展包 p`p1` 中的公共函数不会被自动补全。解决方案：将母扩展包解耦并重命名为 p0`，在 p0` 中用函数 Get 启用子扩展包 p`p1`。可见 Sub-packages and auto-completion。
    • 命名规范。同一类功能的函数应以相同的单词开头命名，判别函数的名称为 xxQ，局部变量的名称全为小写字母，重复模式的名称为单词复数，列表模式的名称为 xxlist。
