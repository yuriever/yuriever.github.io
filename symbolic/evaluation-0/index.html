Expression and evaluation Basic syntax

In this page we list the basic syntax of Mathematica.

Basic syntax

  • Naming: the names of built-in functions start with upper-case letters, hence it's better to name the variables and functions starting from lower-case letters.

  • Pure function: & represents the pure function itself, and #n represents the n-th variable, e.g.

    (#1+#2)&[x,y]

    Out[]= x+y

    Equivalently, 

    Function[{a,b},a+b][x,y]
({a,b}|->a+b)[x,y]

    It's useful to implement differential operators by pure functions like

    op[x_]:=(x D[#,{x,1}])&;
op[x]@F[x]

    Out[]= x F'[x]

  • List vs. function:

    For a list named as h, its elements are accessed by h[[i]], where [[]] is the function Part, to take a part of a list. For multi-level list, the expression h[[i]][[j]] is equivalent to h[[i,j]].

    This approach requires to fill in all the elements when first creating the list, and it is more convenient to set up indexed object whose elements can be filled only in need, via down-value of function h[i]. The multi-level version of down-value is sub-value h[i][j].

    The following table is for managing down-values.

    Syntax Meaning
    h[i]=value add or overwrite a value
    h[i] access a value
    h[i]=. remove a value
    ?h show all defined values
    Clear[h] clear all defined values
    Array[h,n] convert to an explicit list

  • := delayed evaluation, to define functions.

  • /. is the function ReplaceAll, to transform expressions, and can be applied to lists of rules, e.g.

    x/.{{x->x[1]},{x->x[2]}}

    Out[]= {x[1],x[2]}

  • //. is the function ReplaceRepeated, repeated replacing using the rules until the expression fixed. For example,

    gammaRule={
    Gamma[x_]/;x<0->Gamma[1+x]/x,
    Gamma[x_]/;x>1->(-1+x) Gamma[-1+x]
};
Gamma[-5/3]//.gammaRule

    Out[]= 9/10 Gamma[1/3]

  • -> and :> are to define rules, but :> will hold the rules not to be evaluated before using the rules. E.g.

    {x,x,x,x}/.x->RandomReal[]

    will generate one random sample and send it to the list {x,x,x,x}, while,

    {x,x,x,x}/.x:>RandomReal[]

    will generate four samples.

  • /; to set conditions, and the basic usages are,

    pattern/;condition
lhs->rhs/;condition
lhs:=rhs/;condition

  • // is the postfix form of functions with one variable, e.g. expr//FunctionExpand//Simplify.

  • ~ is the infix form of functions with multi variables, e.g. a~f~b is the same as f[a,b].

  • @ is the prefix form of functions with one variable, e.g. f[x] is the same as f@x.

  • @@ is the function Apply, to replace the head of an expression, e.g. f@@{a,b} is the same as f[a,b].

  • /@ is the function Map, e.g.

    f/@{a,b,c,d}

    Out[]= {f[a],f[b],f[c],f[d]}

  • @@@ is the function MapApply, e.g.

    f@@@{{a},{b}}

    Out[]= {f[a],f[b]}

  • @* and /*: left and right compositions.

  • <> and ~~ are the functions StringJoin and StringExpression, to join pieces of strings, e.g.

    Table[Symbol["x"<>ToString[i]],{i,4}]

    Out[]= {x1,x2,x3,x4}
