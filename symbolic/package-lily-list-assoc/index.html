Package lily`list/assoc

扩展包 lily`list 和 lily`assoc 的设计目的是管理数据结构，对应下值和关联两种设计思路。本页面仅详细讨论后者。

Basic ideas

背景动机

有限生成代数由生成元列表 operator 和关系列表 relation 构成，此外在输出结果时，还需格式化表达式的规则列表 printing。多个不同的代数给出不同的列表三元组。

基本框架

由此可抽象出三重结构，类型 type，成员 member，实例 instance。

有如下两种设计思路：

  • 下值 lily`list：以成员为核心，类型为成员的上值，实例为成员的下值。成员的上值用于标记类型、结构和性质，下值用于存储数据。沿此思路得到扩展包 lily`list；

    dataType[member1] ^= type;
dataType[member2] ^= type;

member1[instance1] = {};
member2[instance1] = {};
member1[instance2] = {};
member2[instance2] = {};
...

  • 关联 lily`assoc：用多重关联存储数据，沿此思路得到扩展包 lily`assoc：

    type = <|
    instance1-><|member1->{},member2->{},...|>,
    instance2-><|member1->{},member2->{},...|>,
    ...
|>

应用情景

在有限生成代数的例子中，类型 algebra 包含成员 operator, relation, printing，实例 alg1, alg2, ... 标记不同的代数。其两种实现方式为

  • 下值 lily`list：

    dataType[operator] ^= algebra;
...

operator[alg1] = {};
operator[alg2] = {};
...

  • 关联 lily`assoc：

    algebra = <|
    alg1-><|operator->{},relation->{},printing->{}|>,
    alg2-><|operator->{},relation->{},printing->{}|>,
    ...
|>

函数类型

  • 类型的数据存储：

    • dataType[] 存储全部已定义的类型；
    • dataType[type] 存储类型 type 的数据；
    • dataType[type,member] 存储类型 type 关于成员 member 的默认值。

  • 类型的操作函数为：

    dataDefine, dataProtect, dataUnset

  • 成员的数据结构为：

    listUnsorted, listSorted, setUnsorted, setSorted

  • 实例的操作函数为：

    instanceCheck, instanceUpdate
instanceDefine, instanceDefault, instanceReset, instanceUnset
instanceAdd, instanceDelete

Functionality

Common and default values of instances

dataType[type]["dataInstanceCommon"] stores the common values.

dataType[type,member] stores the default values. After instance operations the default values will be updated.

Checking and updating of instances

Before operating instances check by the function instanceCheck; after operating instances update by the function instanceUpdate. See the section of Instance operation and API

Type and member

  • dataDefine: define and initiate the type.

  • dataProtect: protect against dataUnset. The protected types will not be unset when reloading the package.

  • dataUnset: unset the type. When loading the package, unprotected types will be unset.

Data structure

  • listUnsorted: unsorted list; duplicates allowed;

  • listSorted: sorted list; duplicates allowed;

  • setUnsorted: unsorted set; duplicates not allowed;

  • setSorted: sorted set; duplicates not allowed.

Instance operation

  • instanceCheck

    • instanceCheck["ifTypeNotDefined"]

    • instanceCheck["ifInstanceNotDefined"]

    • instanceCheck["ifInstanceHasDefined"]

    • instanceCheck["ifMemberNotDefined"]

  • instanceUpdate

    • instanceUpdate["updateToDefaultInstance"]: update to default when default, reset, unset, add and delete.

  • instanceDefine

  • instanceDefault

  • instanceReset

  • instanceUnset

    • remove the instances in both the input and default instance list.

  • instanceAdd

  • instanceDelete

API

API of instanceCheck, instanceUpdate, see the definition of instance*`kernel.

instanceCheck[instanceDefine][type,instance];
instanceUpdate[instanceDefine][type,instance];

instanceCheck[instanceDefine][type,instanceList];
instanceUpdate[instanceDefine][type,instanceList];

instanceCheck[instanceReset][type,instance];
instanceUpdate[instanceReset][type,instance];

instanceCheck[instanceUnset][type,instance];
instanceUpdate[instanceUnset][type,instance];

instanceCheck[instanceAdd][type,instance,member,list];
instanceUpdate[instanceAdd][type,instance,member,list];

instanceCheck[instanceDelete][type,instance,member,list];
instanceUpdate[instanceDelete][type,instance,member,list];

API of instanceDefine, instanceDefault, instanceReset, instanceUnset, instanceAdd, instanceDelete: 

instanceDefine[type,instanceList];
instanceDefault[type,instanceList];
instanceReset[type,instanceList];
instanceUnset[type,instanceList];

instanceAdd[type,instanceList,memberRuleOrAssoc];
instanceDelete[type,instanceList,memberRuleOrAssoc];

The memberRuleOrAssoc will be association-ized, hence rule, list of rules, association of rules are accepted.
